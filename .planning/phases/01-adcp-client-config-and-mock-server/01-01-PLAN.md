<objective>
Create the project skeleton (pyproject.toml, package structure, test infrastructure) and implement the YAML config system with dataclass validation. After this plan, a developer can load and validate a config.yaml file containing projector settings and scancode-to-ADCP command mappings.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | ASVS | Mitigation |
|--------|----------|------|------------|
| Malicious YAML (code execution via unsafe load) | HIGH | V5.5 | Use `yaml.safe_load()` exclusively — never `yaml.load()` or `yaml.unsafe_load()` |
| Password stored in plaintext config | LOW | V2.10 | Acceptable for embedded device with physical access only. Document in config comments. |

**Residual risk:** Password in plaintext YAML. Accepted — projector uses unencrypted TCP anyway, and the device is physically secured.
</threat_model>

<action>
Create `src/projector_bridge/errors.py` with the following exception hierarchy:

```python
"""Typed exceptions for ADCP protocol errors."""


class ADCPError(Exception):
    """Base exception for all ADCP errors."""


class AuthError(ADCPError):
    """ADCP authentication failed (err_auth)."""


class CommandError(ADCPError):
    """Unknown ADCP command (err_cmd)."""


class CommandValueError(ADCPError):
    """Invalid parameter value for ADCP command (err_val)."""


class InactiveError(ADCPError):
    """Projector ADCP service inactive, likely in deep standby (err_inactive)."""


class ConnectionError(ADCPError):
    """TCP connection to projector failed (connect/read timeout, refused)."""


class ConfigError(Exception):
    """Configuration file loading or validation error."""
```

Note: `CommandValueError` is used instead of `ValueError` to avoid shadowing the builtin. `ConnectionError` shadows the builtin intentionally within the ADCP namespace — callers import from `projector_bridge.errors`.
</action>

<files>src/projector_bridge/config.py</files>
<action>
Create `src/projector_bridge/config.py` with:

1. Three dataclasses:

```python
@dataclass
class ProjectorConfig:
    host: str
    port: int = 53595
    password: str = ""
    timeout_connect: float = 5.0
    timeout_read: float = 3.0
    retries: int = 3
    retry_delay: float = 0.2

@dataclass
class CommandMapping:
    command: str
    repeat: bool = False
    description: str = ""

@dataclass
class IRConfig:
    device_name: str = "gpio_ir_recv"
    protocol: str = "sony"

@dataclass
class Config:
    projector: ProjectorConfig
    mappings: dict[str, CommandMapping]
    ir: IRConfig
```

2. A `load_config(path: str | Path) -> Config` function that:
   - Opens the file, calls `yaml.safe_load()`
   - Raises `ConfigError("Config file not found: {path}")` if file missing
   - Raises `ConfigError("Config file is empty: {path}")` if yaml returns None
   - Raises `ConfigError("Invalid YAML in config file: {path}: {error}")` on parse error
   - Validates `projector` section exists: `ConfigError("Missing required section: projector")`
   - Validates `projector.host` exists and is non-empty: `ConfigError("Missing required field: projector.host")`
   - Constructs `ProjectorConfig` from `projector` dict with defaults
   - Constructs `CommandMapping` for each entry in `mappings` dict (default: empty dict)
   - Validates each mapping has a `command` field: `ConfigError("Mapping '{key}' missing required field: command")`
   - Constructs `IRConfig` from `ir` dict (default: defaults)
   - Returns `Config` instance

Use `yaml.safe_load()` exclusively — never `yaml.load()`.
</action>
