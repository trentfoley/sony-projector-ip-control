<objective>
Implement the async ADCP TCP client that connects to a projector (or mock), authenticates via SHA256 challenge-response (or NOKEY), sends commands, parses responses, and retries transient connection failures with exponential backoff. After this plan, the client can send any ADCP command and receive typed responses/errors.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | ASVS | Mitigation |
|--------|----------|------|------------|
| Password sent as SHA256 hash over unencrypted TCP | MEDIUM | V2.6 | Matches projector's protocol — no TLS option available. Accepted per protocol constraint. |
| Timing attack on hash comparison (mock server) | LOW | V2.9 | Mock server uses `hmac.compare_digest()` for hash comparison |
| Unbounded read from projector | LOW | V5.5 | `reader.readline()` with `asyncio.wait_for()` timeout prevents hanging on malformed responses |

**Residual risk:** Unencrypted TCP. Accepted — Sony ADCP protocol does not support TLS. Traffic is on a private eth0 link between Pi and projector.
</threat_model>

<action>
Add to `src/projector_bridge/adcp.py`:

```python
async def send_command(config: ProjectorConfig, command: str) -> str:
```

Implementation steps:
1. Connect with `asyncio.wait_for(asyncio.open_connection(config.host, config.port), timeout=config.timeout_connect)`
2. Wrap connection errors: catch `OSError`, `asyncio.TimeoutError` and raise `ConnectionError(f"Failed to connect to {config.host}:{config.port}: {err}")`
3. In try/finally block (finally closes writer):
   a. Read challenge line: `asyncio.wait_for(reader.readline(), timeout=config.timeout_read)`
   b. Decode and strip: `challenge = challenge_line.decode("ascii").strip()`
   c. If challenge is empty: raise `ConnectionError("No challenge received from projector")`
   d. If challenge != "NOKEY":
      - Compute hash: `hashlib.sha256((challenge + config.password).encode()).hexdigest()`
      - Send hash: `writer.write(f"{auth_hash}\r\n".encode("ascii"))`
      - `await writer.drain()`
      - Read auth response, decode, strip
      - If not "ok": raise `AuthError("Authentication failed")`
   e. Send command: `writer.write(f"{command}\r\n".encode("ascii"))`
   f. `await writer.drain()`
   g. Read response line with timeout
   h. Return `parse_response(response.decode("ascii").strip())`
4. Finally block: `writer.close()`, `await writer.wait_closed()`

Key: concatenation order is `challenge + password` (NOT password + challenge). All strings encoded as ASCII. Line terminator is `\r\n`.
</action>

<action>
Add to `src/projector_bridge/adcp.py`:

```python
async def send_command_with_retry(config: ProjectorConfig, command: str) -> str:
```

Implementation:
1. Set `delay = config.retry_delay` (0.2 seconds default)
2. Loop `for attempt in range(config.retries):` (3 attempts default)
3. Try `send_command(config, command)` — return result on success
4. Catch `ConnectionError` only (from projector_bridge.errors, not builtins):
   - If last attempt (`attempt == config.retries - 1`): re-raise
   - Log warning: `"ADCP connection failed (attempt %d/%d), retrying in %.1fs: %s", attempt + 1, config.retries, delay, err`
   - `await asyncio.sleep(delay)`
   - `delay *= 2` (exponential backoff: 200ms -> 400ms)
5. Do NOT catch `AuthError`, `CommandError`, `CommandValueError`, `InactiveError` — these are not retryable

The function signature and return type match `send_command` so callers can swap between them.
</action>
