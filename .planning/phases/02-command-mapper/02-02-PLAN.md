from unittest.mock import AsyncMock, patch
from projector_bridge.config import Config, ProjectorConfig, CommandMapping, IRConfig

    """Build a CommandMapper with test config and mocked ADCP send."""
    if mappings is None:
        mappings = {

    )
    return CommandMapper(config)

**Tests (all async, no decorator needed due to asyncio_mode=auto):**

1. `test_keydown_fires_command` — MAP-02: event_value=1 with mapped non-repeat scancode calls send_command_with_retry exactly once with correct args
2. `test_keyup_ignored` — D-02: event_value=0 never calls send
3. `test_non_repeat_ignores_hold` — MAP-02: event_value=2 with repeat=False mapping does NOT call send
4. `test_repeat_fires_on_hold` — MAP-03: event_value=2 with repeat=True mapping DOES call send
5. `test_unknown_scancode_logged` — MAP-04: unmapped scancode logs at INFO with hex value, no send called
6. `test_rate_limit_drops_rapid_sends` — MAP-05: two handle_scancode calls within 100ms — first fires, second is dropped
7. `test_rate_limit_allows_after_interval` — MAP-05: two calls with >100ms gap — both fire
8. `test_adcp_error_logged_not_raised` — ADCPError in send is caught and logged, does not propagate
9. `test_first_command_always_fires` — first command fires regardless of timing (last_send_time starts at 0)

All tests use `unittest.mock.patch("projector_bridge.mapper.send_command_with_retry", new_callable=AsyncMock)` to mock the ADCP send. Rate limit tests manipulate the event loop time by patching `asyncio.get_event_loop().time()` or by using `asyncio.sleep()` with small intervals.

For rate limit timing tests, use `mapper._last_send_time` manipulation directly:
- Set `mapper._last_send_time = asyncio.get_event_loop().time()` to simulate a just-sent state
- Then immediately call handle_scancode — should be rate-limited
- Then set `mapper._last_send_time = asyncio.get_event_loop().time() - 0.2` — should fire

For the logging test, use `caplog` fixture at INFO level to capture log output and assert "Unknown scancode: 0xFFFFFF" appears.
